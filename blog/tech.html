Test-Driven Development

2/19/2016

This post will focus on a high overview of what Test-Driven Development is, the process to use it, and why it is beneficial.

To get started Test-Driven Development is a software development approach in which a test is written before writing the code. The initially failing test defines what the code should accomplish. Once this is done the developer will write the minimum amount of code to pass that test. Once the test is passed the initial code is refactored to insure it is DRY (don't repeat yourself). This process is repeated as many times as necessary until the overall goal of the code is achieved.

To break the process down a little further there are 6 basic steps to Test-Driven Development:

  • Step 1: Create The Test This is crucial before any code is written. The test is to show a clear understanding of the requirements that need to be accomplished.
  • Step 2: Run Test This is to confirm that the test will not automatically pass without any new code written.
  • Step 3: Write The Code The code does not need to be perfect, it just needs meet the requirements.
  • Step 4: Run The Test This is to insure the initial code passes the specified requirements.
  • Step 5: Refactor The Code This is the time to go back and clean up the code. One should remove any duplication(DRY) and make sure the code is clear.
  • Step 6: Repeat The process is repeated with a new test for the next specification.

This is a beneficial way to write code because using this method allows the developer to focus on writing only the code necessary to pass the test. This in turn translates to cleaner and more readable code. With testing being implemented from the beginning, it can also minimize additional time and money spent to debug later in the software's lifecycle.
